# 总览与设计原则

## 1. 目标

本项目目标是构建一个**前端支持多 ISA、后端 ISA 无关**的处理器模拟框架：

1. 前端完成 ISA 识别与译码，输出统一 `CommonUop`。
2. 后端只处理 `CommonUop`，不感知原始 ISA 细节。
3. 使用精确流水级、统一握手、统一 flush 语义，便于扩展与验证。

## 2. 顶层流水级

### 前端（4 级宏观）

`PIFetch -> IFetch -> PreDecode -> Decode -> FrontEnqueue`

### 后端（6 级）

`Rename -> Dispatch -> Issue -> (Exu | Lsu) -> WriteBack -> Commit`

> 备注：前端可在内部继续细分子阶段；外部接口固定为 4 级口径。

## 3. 设计原则

1. **语义统一优先于性能细节**：先把协议与回滚语义做对，再调性能。
2. **接口稳定优先于模块完备**：先固定 `CommonUop` 与 QM3 同名级间 IO（如 `DecRenIO`、`RenDisIO`）。
3. **组合优先于继承**：前后端共享协议，不强行共享同一个大 IO 类。
4. **可验证性优先**：每一级都有可观测统计和最小测试。
