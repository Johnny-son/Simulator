# 关键组件与模块边界

## 1. 前端组件

### 1.1 `BPU`

- 输入：当前 PC、历史状态、训练更新。
- 输出：预测方向/目标及 meta。

### 1.2 `IFetch/ICache`

- 输入：取指请求。
- 输出：指令块、异常状态。

### 1.3 `PreDecode`

- 输入：指令字节流。
- 输出：切分结果、控制流轻判。

### 1.4 `DecodeRISCV` / `DecodeX86`

- 输入：预译码结果。
- 输出：统一 `CommonUop`。

### 1.5 `FrontEnqueue`

- 输入：`CommonUopPacket`。
- 输出：`fe2be` 通道。

## 2. 后端组件

### 2.1 `UopBuffer`

- 前后端解耦缓冲。
- 对 flush 提供按 `seq_id` 的 younger 清理能力。
- 采用 ring 指针：`head/tail + epoch(flag)`。
- 对外仅暴露 `ChannelBundle` 风格接口，不暴露内部数组。

### 2.2 `Rename`

- RAT/FreeList/ROB 分配。

### 2.3 `Dispatch + IssueQueue`

- 分发与就绪管理。

### 2.4 `Exu`

- 非访存执行通路（INT/BR/CSR 等）。

### 2.5 `Lsu`

- 访存执行通路（Load/Store）。

### 2.6 `WriteBack`

- 汇合 `Exu` 与 `Lsu` 结果并写回。

### 2.7 `ROB + Commit`

- 顺序退休、精确异常、恢复触发。
- 作为全机恢复仲裁点，统一产生 `FlushEvent`。

## 3. 指针所有权边界（新增）

为避免前后端耦合，所有“指针类状态”必须有明确 owner：

1. `UopBuffer`：`head/tail` 仅归 UopBuffer 管理。
2. `ROB`：`enq/deq + epoch` 仅归 ROB 管理。
3. `IQ/LSQ`：各自维护本地 ring 指针，不跨模块修改。

跨模块只传递**索引值**，不传递“可写指针引用”。

## 4. 模块边界规则

1. 前端模块不访问后端内部结构（如 ROB 细节）。
2. 后端模块不依赖 ISA 原始字节编码。
3. 跨边界只允许通过协议化通道和事件总线。
4. 边界字段全部使用 `wire/reg` 位宽类型。
5. 同名字段全局统一语义（`seq_id/rob_idx/br_mask`）。
