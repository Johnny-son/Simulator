# 后端流水级详细设计

## 1. 设计目标

后端仅处理统一 uop，不关心原始 ISA 字节编码；并且与前端共享同一套底层框架（`wire/reg`、握手协议、年龄模型）。

本章采用两层 uop：

1. `FrontUop`：前端输出，表达 ISA 语义。
2. `ExecUop`：后端内部态，补齐资源分配与执行跟踪字段。

## 2. 流水级定义

### 2.1 `Rename`

职责：

- 逻辑寄存器到物理寄存器映射。
- 分配 ROB 项与必要的检查点资源。

输入：`Channel<FrontUopPacket>`（来自 `UopBuffer`）。

输出：`RenamedUopPacket`（内部元素类型为 `ExecUop`）。

### 2.2 `Dispatch`

职责：

- 按功能类型分发到不同 issue queue。
- 维护发射前依赖标记。

输出：`DispatchedUopPacket`。

### 2.3 `Issue`

职责：

- 就绪判断与仲裁。
- 按端口与功能单元能力发射。

输出：`IssuedUopPacket`。

### 2.4 `Exu`

职责：

- 执行非访存类功能（ALU/BRU/CSR 等）。
- 输出执行结果到写回汇合点。

输出：`ExeWbIO`。

### 2.5 `Lsu`

职责：

- 执行访存类功能（Load/Store 相关）。
- 输出访存结果到写回汇合点。

输出：`LsuWbIO`。

### 2.6 `WriteBack`

职责：

- 汇合 `Exu` 与 `Lsu` 的完成结果。
- 写回物理寄存器并向 ROB 提交完成态。
- 广播 wakeup 给等待中的 issue queue 项。

输出：`WbPacket`。

### 2.7 `Commit`

职责：

- 按 ROB 顺序退休。
- 触发精确异常、分支修正、系统状态提交。

输出：`CommitEvent` / `FlushBus`。

## 3. 后端不变量

1. 提交顺序必须与程序顺序一致。
2. Younger 指令不得先于 older 指令提交。
3. flush 后 younger 必须全部失效。
4. 后端对 `isa_tag` 仅做统计，不做功能分支。

## 4. 指针与年龄模型（统一规则）

为对齐启蒙3号后端，所有环形结构（ROB/IQ/LSQ/UopBuffer）采用统一指针语义：

1. 指针状态：`idx + epoch(flag)`。
2. 满空判定：`idx 相等时由 epoch 区分 full/empty`。
3. 年龄比较：禁止直接用裸整数比较，必须调用统一比较函数。

### 4.1 推荐通用比较接口

- `is_younger(a, b, head)`：判断 `a` 是否比 `b` 更新（相对某基准）。
- `in_flush_range(x, flush_point)`：判断条目是否应在本次恢复中被 kill。

### 4.2 ROB 特化规则

ROB 条目携带 `rob_idx + rob_epoch`（或等价 `rob_flag`）：

1. 提交严格按 head 推进。
2. flush 仅清理 younger，older 与已提交不可回滚。
3. 分支误预测回退 tail 时必须跨行处理回卷。

## 5. 完成模型（多源完成掩码）

`ExecUop` 增加：

- `expect_mask`：本条目期望的完成源集合。
- `cplt_mask`：当前已完成源集合。

提交条件：`(cplt_mask & expect_mask) == expect_mask`。

这允许一条指令由多个执行来源共同完成（如 `Exu + Lsu` 辅助状态）。

## 6. 与前端协同点

- 通过 `ready` 背压控制前端灌入速率。
- 通过 `FlushBus` 进行 redirect 与异常恢复。
