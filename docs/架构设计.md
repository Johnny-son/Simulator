# 多ISA模拟器架构设计

## 1.顶层设计
```
CPU
├─ Frontend
│  ├─ Fetch/BTB/TAGE/ICache
│  ├─ DecoderX86
│  ├─ DecoderRISCV
│  └─ UopBuilder（统一产出 CommonUop）
├─ Backend（ISA-agnostic）
│  ├─ Rename
│  ├─ Dispatch
│  ├─ IssueQueue
│  ├─ Exu 
|  ├─ Lsu
│  ├─ Rob
│  └─ Commit
├─ MemSubsystem
└─ Verification
   ├─ DifftestChecker
   ├─ ModuleBridge(RTL co-sim)
   └─ Trace/Replay
```

### 1.1 前后端边界
- 前端负责：ISA 指令译码、宏指令拆分、语义补全。
- 后端负责：rename/dispatch/issue/execute/retire。

## 2.统一 uOP 契约

### 2.1 最小字段
```cpp
struct CommonUop {
  // 追踪与聚合
  uint64_t inst_seq;      // 宏指令序号
  uint64_t pc;
  uint8_t  macro_id;
  bool     is_microop;
  bool     is_last_microop;

  // 操作语义
  UopKind  kind;          // ALU/BR/LOAD/STORE/SYS/FP/VEC...
  uint8_t  width;         // 8/16/32/64/128/...
  bool     is_signed;

  // 寄存器操作数（架构寄存器，rename 前）
  uint16_t src_areg[3];
  uint16_t dst_areg[2];
  uint8_t  src_num;
  uint8_t  dst_num;

  // 立即数/寻址
  uint64_t imm;
  MemAttr  mem_attr;      // BaseImm / BaseIndexShift / PCRel / PrePost

  // 控制与异常提示
  CtrlFlag ctrl;          // branch/call/ret/indirect/fence
  ExceptHint except_hint;

  // ISA 元信息（仅前端与提交层消费）
  IsaTag   isa;           // RV64 / X86_64 / ...
  uint32_t isa_opcode;
  uint64_t isa_aux0;      // e.g. x86 prefix bits / rv csr index
};
```

### 2.2 关键约束

1. 后端禁止读取 `isa_opcode` 决策功能；只看 `kind/mem_attr/ctrl`。
2. `is_last_microop=true` 的提交点才触发宏指令级 difftest。
3. 复杂指令允许走 slow-path（microcode/assist），不要污染通用快路径。
