# 多 ISA 模拟器架构设计

## 架构总览

- 前端：`PIFetch -> IFetch -> PreDecode -> Decode -> FrontEnqueue`
- 后端：`Rename -> Dispatch -> Issue -> (Exu | Lsu) -> WriteBack -> Commit`
- 核心目标：前端多 ISA 译码，后端 ISA 无关执行。

## 当前实现状态（2026-04）

1. 顶层采用 `SimCpu -> FrontTop + BackTop` 的分层驱动。
2. 前端已按 5 级级联：`PIFetch -> IFetch -> PreDecode -> Decode -> FrontEnqueue`。
3. 后端执行路径已拆分为独立 `Exu` 与 `Lsu` 模块，并在 `WriteBack` 汇合。
